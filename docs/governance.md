# Governance: AI Credit Risk Analyst

## 1. Реестр рисков (Risk Register)

| Риск | Вероятность | Влияние | Обнаружение | Защита | Остаточный риск |
|------|-------------|---------|-------------|--------|-----------------|
| **Неправильная рекомендация** — агент даёт ошибочный approve/reject | Средняя | Критическое | Ревью аналитиком, A/B тесты, метрики корректности | Человек в цикле, финальное решение только за аналитиком | Средний |
| **Утечка персональных и финансовых данных** | Низкая | Критическое | Аудит логов, мониторинг доступа | Шифрование, минимизация данных в промптах, политика хранения | Низкий |
| **Галлюцинации LLM** — некорректные факты в объяснении | Средняя | Высокое | Ревью аналитиком, проверка на тестовой выборке | Чёткие промпты, ограничение домена, привязка к структурированным данным | Средний |
| **Prompt injection** — манипуляция агентом | Средняя | Высокое | Анализ входных данных, алерты на аномалии | Санитизация ввода, изоляция контекста, подтверждение действий | Средний |
| **PD-модель не отвечает** | Низкая | Среднее | Мониторинг latency, health checks | Retry, fallback на rule-based оценку, graceful degradation | Низкий |
| **Некорректная интерпретация policy** | Средняя | Высокое | Метрика точности policy check, ревью | RAG с чёткими правилами, валидация вывода | Средний |
| **Превышение бюджета на LLM API** | Средняя | Среднее | Дашборды затрат, лимиты | Rate limiting, кэширование, бюджеты по окружениям | Низкий |

## 2. Политика логирования

### 2.1 Что логируется

| Категория | Данные | Срок хранения | Назначение |
|-----------|--------|---------------|------------|
| **Запросы** | ID сессии, тип запроса, timestamp, client_id (хеш/обезличенный), сумма кредита | 90 дней | Отладка, аналитика использования |
| **Ответы агента** | ID, тип, timestamp, длина ответа, decision (approve/reject/review), флаги | 90 дней | Аудит, качество |
| **Действия** | Подтверждённые действия, отказы, эскалации | 1 год | Compliance, расследование инцидентов |
| **Ошибки** | Текст ошибки, stack trace (без ПДн), контекст | 90 дней | Диагностика |
| **Метрики** | Latency, токены, стоимость вызова, policy check result | 1 год | Оптимизация, бюджеты |

### 2.2 Что не логируется

- Полные тексты заявок с персональными данными (ФИО, паспорт, адрес) в открытом виде
- Содержимое кредитной истории в логах (допускается агрегаты: кол-во записей, сумма долга)
- Пароли, токены доступа, API-ключи

### 2.3 Формат и доступ

- Логи в структурированном формате (JSON)
- Доступ по ролям: операторы — ошибки и метрики; аудиторы — полный доступ к аудит-логам
- Ротация и архивация согласно политике хранения данных организации

## 3. Работа с персональными данными (ПДн)

| Принцип | Реализация |
|---------|------------|
| **Минимизация** | В промпты передаются только поля, необходимые для анализа риска; идентификаторы — по возможности обезличенные |
| **Цель** | Анализ кредитного риска; данные не используются для иных целей без согласия |
| **Хранение** | ПДн в логах не хранятся; при необходимости — псевдонимизация или агрегация |
| **Передача** | LLM API — только через защищённые каналы; проверка DPA с провайдером (если облачный) |
| **Права субъектов** | Процедуры для запросов на доступ, исправление, удаление в соответствии с 152-ФЗ |
| **Финансовые данные** | Кредитная история, скоринговые баллы — обрабатываются с учётом конфиденциальности |

## 4. Защита от injection

| Мера | Описание |
|------|----------|
| **Санитизация ввода** | Валидация и экранирование пользовательского ввода; отклонение подозрительных паттернов |
| **Разделение контекста** | Чёткое разделение системного промпта, данных заявки и пользовательского запроса |
| **Ограничение домена** | Агент отклоняет запросы вне темы кредитного риска |
| **Длина ввода** | Ограничение размера пользовательского сообщения |
| **Мониторинг** | Алерты на аномальные запросы (длина, структура, ключевые слова инъекций) |
| **Тестирование** | Регулярные red-team проверки на уязвимости |

## 5. Подтверждение действий

| Тип действия | Подтверждение | Описание |
|--------------|---------------|----------|
| **Рекомендация по заявке** | Не требуется | Агент только рекомендует; решение принимает аналитик |
| **Запрос кредитной истории** | Автоматически | Внутренний вызов API; логирование |
| **Вызов PD-модели** | Автоматически | Внутренний вызов; логирование |
| **Экспорт / выгрузка данных** | Подтверждение аналитиком + аудит | Явное действие пользователя |
| **Изменение настроек агента** | Только для админов | Ролевая модель |
| **Критичные внешние вызовы** | Подтверждение в UI | Для операций, влияющих на внешние системы |

### Принципы

- Агент **не выполняет** необратимые действия без подтверждения пользователя
- Финальное решение по выдаче кредита принимает только человек
- Все действия логируются с привязкой к пользователю и сессии
