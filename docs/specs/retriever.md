# Spec: Retriever (policy RAG)

## Назначение

Поиск релевантных фрагментов **кредитной политики банка** для проверки соответствия и для контекста LLM (с цитированием).

## Источники

| Источник | Формат | Версионирование |
|----------|--------|-----------------|
| Документы политики | Markdown / фрагменты PDF→text | Поле `policy_version` в метаданных чанка |

## Индекс

- **Chunking:** 512–1024 токенов, overlap 10–15%.
- **Embeddings:** одна модель на весь PoC (фиксировать в config); размерность и нормализация — по документации модели.
- **Хранилище:** векторное (например, Chroma / локальный индекс) + опциональный кэш embedding для повторяющихся запросов.

## Поиск

1. **Query builder** формирует запрос из state: `loan_amount`, флаги (`thin_file`, `pd_bucket`), ключевые слова из правил.
2. **Vector search:** top-k (k=5 по умолчанию), минимальный порог similarity; если ниже — пустой результат.
3. **Reranking (опц.):** cross-encoder или LLM-rerank на top-k → top-3 в контекст LLM.

## Ограничения

- Максимум **N токенов** политики в одном промпте LLM (см. `memory-context.md`).
- Не подмешивать в retrieval чужие документы (одна коллекция «policy» в PoC).

## Выходной контракт

```json
{
  "chunks": [
    {
      "chunk_id": "string",
      "policy_version": "string",
      "title": "string",
      "text": "string",
      "score": 0.0
    }
  ],
  "fallback_used": false,
  "retrieval_mode": "vector | bm25 | empty"
}
```

## Ошибки и fallback

| Условие | Действие |
|---------|----------|
| Пустой vector | Включить BM25 по тем же чанкам |
| Снова пусто | `empty` + rules engine только по жёстким правилам; флаг в отчёте |
