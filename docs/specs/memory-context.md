# Spec: Memory / Context

## Технологии (реализация PoC)

| Аспект | Технология |
|--------|------------|
| Область жизни state | Один HTTP-запрос к `POST /api/assess`; без серверной сессии Next.js |
| Хранилище сессии | Не используется: нет Redis, нет Supabase для state |
| Контекст LLM | Строки и объекты в стеке вызова `runAssessment`; персистентность между запросами отсутствует |

## Session state

Структура (логическая), в рамках одной обработки запроса:

| Поле | Описание |
|------|----------|
| `session_id` | В PoC не генерируется; можно добавить UUID в заголовке ответа |
| `slots` | `client_id`, `loan_amount` из `parseSlots` |
| `credit_history_result` | Результат tool или ошибка |
| `pd_result` | Результат tool или fallback |
| `policy_rules_outcome` | Итог `applyRules` |
| `retrieval_chunks` | Результат `retrievePolicy` |
| `draft_report` | Промежуточные `facts` перед LLM |

**TTL:** один запрос; долговременное хранение PII не выполняется.

## Memory policy

- **Долговременная память** о клиентах: **не используется** в PoC.
- История диалога: в UI одно поле сообщения; многоходовый чат не реализован (при расширении — последние K≤10 реплик в теле запроса).

## Context budget (LLM)

| Компонент | Ограничение |
|-----------|-------------|
| Системный промпт | Фиксированный шаблон в `lib/llm/explain.ts` |
| Факты (facts block) | JSON-сериализация агрегатов без сырых записей БКИ |
| Чанки политики | `retrieverTopForLlm` (по умолчанию 3) |
| User message | До 8000 символов (pre-guardrail) |

При превышении budget: retry с одним чанком политики (`generateExplanationWithRetry`).

## Конфиденциальность

В промпт LLM не передавать больше ПДн, чем нужно для объяснения (см. `docs/governance.md`): предпочитать агрегаты и флаги.
