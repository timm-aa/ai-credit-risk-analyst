# Spec: Memory / Context

## Session state

Структура (логическая), хранится в рамках одной сессии обработки запроса (и короткого диалога уточнений):

| Поле | Описание |
|------|----------|
| `session_id` | UUID |
| `slots` | `client_id`, `loan_amount`, опц. |
| `credit_history_result` | Результат tool или ошибка |
| `pd_result` | Результат tool или ошибка / degraded |
| `policy_rules_outcome` | Итог rules engine |
| `retrieval_chunks` | Список чанков + ids |
| `draft_report` | Промежуточный структурированный отчёт |

**TTL:** минуты; без персистентности PII между сессиями в PoC.

## Memory policy

- **Долговременная память** о клиентах: **не используется** в PoC.
- История диалога: ограничить **последними K** сообщениями (K≤10) для уточнений.

## Context budget (LLM)

| Компонент | Ограничение |
|-----------|-------------|
| Системный промпт | Фиксированный шаблон, без лишних примеров |
| Факты (facts block) | Сжатый JSON/таблица, без дублирования полных ответов БКИ |
| Чанки политики | Top-3 после rerank или top-5 без rerank; **жёсткий лимит токенов** |
| User message | Макс. длина символов (например, 4–8K) |

При превышении budget: урезать чанки политики с наименьшим score, затем сокращать facts до ключевых полей.

## Конфиденциальность

В промпт LLM не передавать больше ПДн, чем нужно для объяснения (см. `governance.md`): предпочитать агрегаты и флаги.
